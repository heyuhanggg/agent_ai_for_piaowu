# 运维SOP标准操作流程

---

## 服务发布与回滚SOP

### 发布前检查
1. 确认代码已通过CI流水线（单元测试、集成测试全部通过）
2. 确认数据库DDL变更已提前执行并验证
3. 确认配置变更已同步到配置中心
4. 确认发布窗口已通知相关团队
5. 准备好回滚方案和回滚包

### 发布流程
1. 灰度发布：先发布1个实例，观察5分钟
2. 检查灰度实例的错误日志、响应时间、成功率
3. 灰度无异常后，分批发布剩余实例（每批间隔2分钟）
4. 全量发布完成后，持续观察15分钟
5. 确认无异常后，发布完成

### 回滚流程
1. 发现异常后立即暂停发布
2. 执行回滚命令，恢复到上一个稳定版本
3. 回滚完成后检查服务状态和核心指标
4. 通知相关团队回滚情况
5. 记录回滚原因，排查问题后再次发布

### 回滚触发条件
- 错误率上升超过基线值的2倍
- P99延迟上升超过基线值的3倍
- 出现未预期的异常类型
- 核心业务接口不可用

## 紧急扩容SOP

### 触发条件
- CPU使用率持续超过80%达5分钟
- 内存使用率持续超过85%达5分钟
- 接口P99延迟持续超过预期值的2倍
- 请求队列积压持续增长

### 操作流程
1. 确认瓶颈服务（通过监控指标定位）
2. 评估需要扩容的实例数量（当前流量/单实例承载量）
3. 执行水平扩容（新增服务实例）
4. 等待新实例启动并加入负载均衡（约2-3分钟）
5. 观察扩容效果（CPU、延迟、错误率是否下降）
6. 如果效果不明显，排查是否为下游瓶颈（数据库、Redis等）

### 注意事项
- 扩容前确认基础资源（服务器/容器）充足
- 扩容后需同步检查下游依赖是否能承受增加的流量
- 高峰过后及时缩容，避免资源浪费
- 记录扩容事件，后续分析容量规划

## 数据库慢SQL应急处理SOP

### 发现方式
- 监控告警：慢SQL数量突增
- 服务日志：接口超时增多
- 数据库监控：活跃连接数飙升

### 处理流程
1. 登录数据库，查看当前活跃查询（SHOW PROCESSLIST）
2. 找出执行时间最长的SQL语句
3. 评估影响范围（涉及哪些服务和接口）
4. 对于明显有问题的SQL：
   - 如果是全表扫描：KILL掉该查询，通知开发添加索引
   - 如果是锁等待：找出持锁的事务，评估是否可以KILL
   - 如果是大批量操作：KILL后改为分批执行
5. 持续观察数据库连接数和响应时间恢复情况

### 预防措施
- 所有SQL上线前必须EXPLAIN审查
- 配置慢SQL监控告警（阈值：1秒）
- 定期巡检慢SQL日志，推动优化
- 大表操作（DDL、批量更新）安排在低峰期执行

## Redis故障应急SOP

### 故障类型1：Redis主节点宕机
1. 确认哨兵是否自动完成主从切换
2. 如果自动切换成功，检查应用是否已连接到新主节点
3. 如果自动切换失败，手动执行主从切换
4. 恢复原主节点为从节点
5. 检查数据一致性

### 故障类型2：Redis内存告警
1. 查看Redis内存使用详情（INFO MEMORY）
2. 找出大key（redis-cli --bigkeys 或 MEMORY USAGE）
3. 评估是否可以清理过期/无用数据
4. 临时调大maxmemory（如果服务器内存允许）
5. 排查内存增长原因，制定长期优化方案

### 故障类型3：Redis性能下降
1. 查看Redis慢日志（SLOWLOG GET 20）
2. 检查是否有大key操作或危险命令（KEYS、SMEMBERS）
3. 查看客户端连接数是否过多
4. 检查是否正在进行RDB/AOF持久化操作
5. 检查网络延迟

## 服务健康巡检SOP

### 每日巡检项目
1. **服务可用性**：检查所有微服务实例是否正常运行
2. **错误日志**：查看最近24小时ERROR级别日志数量和趋势
3. **接口性能**：检查核心接口P99延迟是否在正常范围
4. **资源使用**：检查CPU、内存、磁盘使用率
5. **JVM状态**：检查GC频率和耗时、堆内存使用率
6. **数据库**：检查慢SQL数量、连接数、主从同步延迟
7. **Redis**：检查内存使用率、连接数、命中率
8. **消息队列**：检查消息积压数量

### 巡检阈值标准
| 指标 | 正常范围 | 告警阈值 | 严重阈值 |
|------|---------|---------|---------|
| CPU使用率 | <60% | >70% | >85% |
| 内存使用率 | <70% | >80% | >90% |
| 磁盘使用率 | <70% | >80% | >90% |
| GC暂停时间 | <200ms | >500ms | >1000ms |
| 接口P99延迟 | <500ms | >1000ms | >3000ms |
| 错误率 | <0.1% | >1% | >5% |
| 数据库连接使用率 | <60% | >80% | >95% |
| Redis内存使用率 | <70% | >80% | >90% |
