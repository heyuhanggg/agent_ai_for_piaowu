# 微服务架构与依赖关系

---

## 服务拓扑总览

### 核心服务列表
| 服务名 | 端口 | 职责 | 核心依赖 |
|--------|------|------|---------|
| gateway-service | 6085 | API网关，请求路由、鉴权、限流 | Redis、各下游服务 |
| user-service | 6086 | 用户管理、登录注册、实名认证 | MySQL、Redis |
| program-service | 6087 | 节目管理、查询、推荐 | MySQL、Redis、Elasticsearch |
| order-service | 6088 | 订单创建、查询、状态管理 | MySQL、Redis、MQ |
| pay-service | 6089 | 支付流程、回调处理 | MySQL、Redis、第三方支付 |
| base-data-service | 6090 | 城市、场馆等基础数据 | MySQL、Redis |
| admin-service | 6091 | 后台管理 | MySQL |
| customize-service | 6092 | 定制服务 | MySQL、Redis |

## 服务调用关系

### 购票核心链路
```
用户请求 → gateway-service → program-service（查询节目）
                            → program-service（查询票档）
                            → user-service（获取用户信息）
                            → order-service（创建订单）
                            → pay-service（发起支付）
                            → pay-service（支付回调）→ order-service（更新状态）
```

### 依赖关系说明
- **gateway-service** 是所有请求的入口，依赖所有下游服务
- **order-service** 依赖 user-service（校验用户）、program-service（校验节目和票档）
- **pay-service** 依赖 order-service（更新订单状态）
- **program-service** 依赖 base-data-service（获取城市和场馆信息）

### 关键依赖影响分析
| 如果此服务故障 | 直接影响 | 间接影响 |
|--------------|---------|---------|
| gateway-service | 所有请求不可用 | 全站不可用 |
| user-service | 登录、注册、实名认证 | 无法下单（需要用户信息） |
| program-service | 节目查询、推荐 | 无法下单（需要节目信息） |
| order-service | 下单、查询订单 | 支付回调无法更新状态 |
| pay-service | 支付流程 | 订单无法完成支付 |
| MySQL | 所有写操作和无缓存的读操作 | 几乎全站不可用 |
| Redis | 缓存失效、Session丢失、限流失效 | 数据库压力飙升 |

## 关键配置说明

### 数据库配置建议
- **连接池**：maxPoolSize建议设置为 CPU核数 * 2 + 磁盘数，通常30-50
- **超时时间**：connectionTimeout=30s，idleTimeout=600s，maxLifetime=1800s
- **慢SQL阈值**：建议1秒，超过1秒的SQL需要优化

### Redis配置建议
- **连接池**：maxTotal=50，maxIdle=20，minIdle=5
- **超时时间**：connectTimeout=2s，readTimeout=2s
- **序列化**：推荐使用Jackson2JsonRedisSerializer
- **Key命名规范**：{业务}:{模块}:{标识}，如 ticket:program:detail:123

### JVM配置建议
- **堆内存**：-Xms和-Xmx设置相同，避免动态扩容带来的GC压力
- **推荐配置**：-Xms2g -Xmx2g -XX:+UseG1GC -XX:MaxGCPauseMillis=200
- **必须配置**：-XX:+HeapDumpOnOutOfMemoryError -XX:HeapDumpPath=/logs/heapdump/

## 告警规则配置

### 核心告警规则
| 告警名称 | 条件 | 级别 | 通知方式 |
|---------|------|------|---------|
| 服务宕机 | 实例健康检查连续3次失败 | P0 | 电话+短信+钉钉 |
| 错误率飙升 | 5分钟内错误率>5% | P0 | 电话+短信+钉钉 |
| 接口延迟 | P99延迟>3秒持续5分钟 | P1 | 短信+钉钉 |
| CPU过高 | CPU>85%持续5分钟 | P1 | 短信+钉钉 |
| 内存过高 | 内存>90%持续5分钟 | P1 | 短信+钉钉 |
| 磁盘空间 | 磁盘>90% | P1 | 短信+钉钉 |
| 慢SQL | 单SQL>5秒 | P2 | 钉钉 |
| GC异常 | Full GC>3次/分钟 | P2 | 钉钉 |
